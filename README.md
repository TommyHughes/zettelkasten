# Tommy's Zettelkasten

This is where I take notes I'd like to keep. To give this zettelkasten the room that it needs to grow organically, only minimal formatting requirements will be required. The idea is to reduce formatting and linking fatigue. Don't worry if ideas don't get linked up like they should. Just do your best to take good notes, and add meaningful meta data for lookup.

## Formatting

Files will *affectionately* be called "zettels".

Zettels are typically .md files. Sometimes other file types are needed and might be considered a zettel, but these would be exceptions (like a .pdf with a diagram of some sort).

There should be a preference on giving each zettel a readable name, but it is not required.

Each (typical) zettel requires a short YAML front matter section with the following tags,

**type:** What type of zettel is this?

**date:** The most recent update associated with the notes.

**tags:** key words associated with the zettel. Keep tenses simple but don't torture the word to do so.

These are the only required items for the content of any zettel. Everything else if free-form.

While the three tags included above are required, additional tags can be added when it's deemed appropriate. Front matter YAML is extensible.
